01. Definition

専門家の目線による
「提案の妥当性」を自動診断。

AI.BUDDY の技術的役割

AI.BUDDYは、商談における音声・視覚データを構造化し、外部ドメインナレッジと突合させることで、「その提案に勝算はあるか」をプロの視点で評価する診断システムである。 単なる要約ではなく、特定の業界慣習や商流に基づいた「専門家としての意見」を提供し、営業担当者の提案品質を客観的に検証するパートナーとして動作する。

02. Processing Flow

情報の特定からフィードバック生成まで

1. 顧客情報の特定と属性情報の注入

gBizINFO API を介して法人番号を特定し、AIに「業界の常識」を同期する。

AIは取得データに基づき、その業界特有の商流や「刺さるポイント」を網羅した評価基準をその場で生成する。

2. 商談の動画やテキストデータからの多角的解析

Gemini 1.5 Proにより、動画・音声を直接入力。表情の硬さ、頷き、発話のラグなどの「非言語情報」を直接読み取り、納得感や懸念点を抽出する。

3. フィードバック精度向上のための対話型検証

AIは「商談ログからは読み取れないが、成約に不可欠なピース」を検出し、ユーザーへの逆質問を行います。

想定される検証クエリのサンプル

「gBizINFOのデータから、お相手は現在○○への注力が見受けられます。提案中、その戦略目標への貢献を問うような踏み込んだ反応はありましたか？」

「価格提示後に約4秒間の沈黙が検出されました。お相手の表情からは迷いが見えましたが、その際、言葉にならなかった懸念点などは感じ取れましたか？」

「決裁ルートの把握が不十分な可能性があります。本日不在だったキーマンへの説明順序や、合意形成の段取りについて具体的に踏み込めましたか？」

03. Analysis Logic

専門家の思考を模倣する論理設計

内部レビュー型エージェント設計

【Role 01：評価】

業界知識を背景に、今回の提案が商流や課題に照らして妥当かをプロの視座で診断する。

【Role 02：批判】

顧客の立場になりきり、提案の「納得できないロジック」を厳しく指摘し、Role 01を修正させる。

【Role 03：統合】

双方の議論を整理し、証拠（発言引用）を確認. 現場で即座に役立つ改善のヒントを確定する。

04. Output Definition

最終フィードバックのアウトプット定義

要約
商談の核となる論点と、お客様の真の関心事を構造化。
評価
専門家から見た、今回の提案の「筋が良い」点。
改善
業界の観点で不足していた情報や、お客様が納得しきれなかった要素の指摘。
一手
次回の面談で、提案の精度を劇的に高める具体的なアドバイス。
05. Prerequisites & Tasks

開発着手に向けた必須準備事項

システムの構築を開始するためには、以下の「外部サービスへの申請」および「環境設定」が完了している必要があります。これらは外部要因による待ち時間が発生するため、最優先で着手すべきブロック事項です。

Waiting for Application

1. gBizINFO API 利用環境の構築

経済産業省が提供する法人情報検索APIを利用するための準備です。APIキーが発行されるまで、法人検索機能の実装・テストは行えません。

必要なアクション
  • gBizID（エントリーまたはプライム）の取得。
  • 「gBizINFO API利用申請」フォームよりAPIキー発行を依頼。
所要時間の目安

APIキーの発行には、申請から数営業日〜1週間程度を要する場合があります。開発遅延を防ぐため、即時の申請を推奨します。

Admin Settings Required

2. Google Workspace & API 認証設定

GWS版GeminiをAPI経由で安全に利用するための、特権管理者による設定作業です。

必要なアクション
  • 管理コンソールでの「Google AI Studio」の利用許可設定。
  • APIキーの生成と GAS への環境変数登録。
