01. Definition

専門家の目線による
「提案の妥当性」を自動診断。

AI.BUDDY の技術的役割

AI.BUDDYは、商談における音声・視覚データを構造化し、外部ドメインナレッジと突合させることで、「その提案に勝算はあるか」をプロの視点で評価する診断システムである。 単なる要約ではなく、特定の業界慣習や商流に基づいた「専門家としての意見」を提供し、営業担当者の提案品質を客観的に検証するパートナーとして動作する。

02. Processing Flow

顧客特定から専門家フィードバックまで

1. 顧客情報の特定と業界視点のロード

gBizINFO APIから取得した属性データをもとに、AIに「お相手が属する業界の常識」を同期します。

【技術解説：評価基準の動的生成】

AIは、取得した事業内容に基づき「この業界であれば、何を評価基準に据えるべきか」というチェックリストをその場で生成します。これにより、お客様の業種ごとに異なる「刺さるポイント」や「懸念すべき観点」を網羅した評価が可能になります。

2. 商談の動画やテキストデータからの多角的解析

Gemini 1.5 Proにより、提案に対するお客様の反応をダイレクトに読み取ります。

非言語情報の解析によって特定される「本音」
提案時の視覚反応

スライド提示時や見積もり提示時の表情の硬さや頷きを解析し、「納得感」を数値化します。

話者間のラグとトーン

質問に対するお客様の回答の「迷い（間）」や、声のトーンの変化から、提案に対する警戒心や懸念点を抽出します。

3. フィードバック精度向上のための対話型検証

AIが「この提案には、まだ確認すべき懸念がある」と判断した場合、ユーザーに逆質問を行い、より確かな評価へと昇華させます。

「業界の慣習上、○○の要件は必須となることが多いですが、先方からそのような要望や言及はありましたか？」

03. Analysis Logic

専門家の思考を模倣する論理設計

内部検証エージェント・ループ設計

一人のAIの独走を防ぎ、多角的な「意見」を生成するため、内部で3つの役割を戦わせる設計を採用しています。これは現時点の最善策であり、改善を続けていきます。

Role 01：評価者

【業界エキスパート】

業界知識に基づき、今回の提案が「お客様の商流や課題」に照らして妥当かどうかを判定する。

Role 02：批判者

【顧客側レビュワー】

顧客の立場になりきり、提案の「弱い部分」や「納得できないロジック」を容赦なく指摘する。

Role 03：統合者

【フィードバック監修】

双方の意見を整理し、現場で使える「改善のヒント」として最終的なフィードバックを確定する。

/* 最善を追求するための自己検証プロセス */ { "検証原則 01": "証拠優先：すべての意見は面談中の具体的な発言や反応を根拠としなければならない", "検証原則 02": "多角的視点：エキスパートとしての『正論』と、顧客レビュワーとしての『感情的な納得感』を両立させる", "検証原則 03": "具体的指摘：『良い良くない』の判定に留まらず、業界観点を踏まえた具体的な改善ヒントを付与する" }
04. Output Definition

最終フィードバックのアウトプット定義

要約
商談の核となる論点と、お客様の真の関心事を整理。
評価
専門家から見た、今回の提案の「筋が良い」点と有効だった行動。
改善
業界の観点から見て不足していた情報や、お客様が納得しきれなかった要素の指摘。
一手
次回の面談で、提案の質を劇的に高めるための具体的なアドバイス。
